1. Media Queries Description: Use CSS media queries to apply styles based on device characteristics such as screen width, height, orientation, and resolution. Example: @media (max-width: 600px) { body { background-color: lightblue; } } 2. Flexible Layouts with Percentages Description: Use relative units like percentages (%) or viewport units (vw, vh) for widths and heights instead of fixed units like px. Example: .container { width: 80%; /* Adapts to the width of the viewport */ margin: auto; } 3. CSS Grid Layout Description: Create flexible grid-based layouts that adjust automatically to screen sizes. Example: .grid-container { display: grid; grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); gap: 10px; } 4. CSS Flexbox Description: Use Flexbox for creating fluid, flexible layouts that align items dynamically. Example: .flex-container { display: flex; flex-wrap: wrap; justify-content: space-around; } .flex-item { flex: 1 1 200px; /* Flex-grow, flex-shrink, and base size */ } 5. 9. Mobile-First Design Description: Start with the smallest screen size styles (default styles) and use media queries for larger screens. Example: body { font-size: 16px; /* Default for mobile */ } @media (min-width: 768px) { body { font-size: 18px; /* Larger screens */ } } 10. Responsive Tables Description: Use techniques like horizontal scrolling, collapsing columns, or stacking rows for making tables responsive. Example: .table-container { overflow-x: auto; } 11. Aspect Ratio Description: Maintain the aspect ratio of elements like images and videos using CSS. Example: .responsive-video { aspect-ratio: 16 / 9; width: 100%; } 12. Testing and Tools Description: Test responsiveness with tools and emulators. Browser DevTools: Use the "Responsive Design Mode" in Chrome or Firefox. Tools: Services like Google Mobile-Friendly Test or Responsinator. Best Practices Start Mobile-First: Design for small screens first and progressively enhance for larger devices. Keep Layouts Fluid: Avoid fixed widths and heights. Test Across Devices: Check on actual devices or use emulators. Prioritize Performance: Optimize assets, lazy load images, and minimize CSS and JavaScript.
